Weekend Rustic Ray Tracing - Part 2

Introduction

If you're at all curious about ray tracing, I recommend starting with Peter Shirley's Ray Tracing Minibooks. In this series of posts, I'll be converting the code in Ray Tracing in One Weekend (Book 1 in the series) to Rust. Rather than rewrite Mr. Shirley's text, I'll be explaining my implementation choices.

Source code for this series is in this repository. Each chapter has its own branch: Chapter01, Chapter02, etc. Each chapter branch is the completed code for that chapter. The master branch will be the final renderer from the end of the book.

This little project will then be followed with some modifications to allow the code to render in a web browser.

Chapter 2: The vec3 class

Rust doesn't do classes, so we use a struct to hold the data for our three-dimensional vector. Vec3's methods match the methods from C++, with a minor exception: I've decided that length() should rely on squared_length() to keep the code DRY. Finally, we write operations for our vector using method implementations on each named operator.

This transliteration is fairly straightforward, but I've made some modifications. I don't care for the default initialization syntax. This is easily remedied with another new method taking an array of three items, or three f64 arguments. Also, since we're using it to hold RGB values, rather than use the index operator on the vector, I'm calling the color component methods.

There's one more change I've made: rather than multiply each component by 255.99, I'm scaling the vector and then casting the components to u8. Not only do we repeast less code, but we also make use of the operators we implemented.
